# Парсинг Telegram-каналов и анализ упоминаний брендов

Инструмент для скачивания постов из публичных Telegram-каналов и анализа упоминаний брендов На выходе — отчёты с тональностью по каждому автору.

## Установка

```bash
pip install -r requirements.txt
```

Для скачивания постов нужны Telegram API-ключи (`api_id` и `api_hash`) — их можно получить на [my.telegram.org](https://my.telegram.org). При первом запуске нужно будет ввести номер телефона и код из Telegram. Дальше авторизация сохраняется и повторный ввод не нужен.

## Как пользоваться

Весь пайплайн управляется через чат с агентом в Cursor. Просто описываете задачу обычным языком.

### 1. Скачать посты из каналов

Примеры запросов агенту:

> Скачай посты из канала android_broadcast с октября 2025

> Скачай посты из каналов mobiledevnews, ios_broadcast, kotlin_adept с 1 октября 2025

Агент сам запустит скрипт `scrape.py`, посты сохранятся в папку `exports/`.

### 2. Построить отчёт

> Запусти анализ каналов

Агент запустит `analyze_channels.py`, который найдёт все скачанные каналы, найдёт упоминания брендов и создаст:

- `reports/report.md` — сводный отчёт по всем каналам
- `reports/by_author/` — отдельный файл по каждому автору

При первом запуске тональность постов ещё не определена — в отчётах будет пометка «не размечено».

### 3. Разметить тональность

> Разметь тональность по очереди из artifacts

Агент прочитает посты, оценит отношение автора к каждому бренду и сохранит результат.

Если постов много — результат разбивается на части. В этом случае попросите агента:

> Склей батчи тональности

### 4. Пересобрать отчёт с тональностью

> Запусти анализ каналов ещё раз

Теперь отчёты будут содержать оценку тональности для каждого упоминания.

## Порядок действий коротко

1. «Скачай посты из канала X с такой-то даты»
2. «Запусти анализ»
3. «Разметь тональность»
4. «Запусти анализ ещё раз»
5. Готовые отчёты — в папке `reports/`

## Что делает каждый файл

| Файл | Что делает |
|------|------------|
| `scrape.py` | Скачивает посты из Telegram-каналов в `exports/` |
| `analyze_channels.py` | Анализирует посты и строит отчёты в `reports/` |
| `content_enrich.py` | Подтягивает текст из YouTube-роликов, если в посте есть ссылка (запускается автоматически) |
| `merge_sentiment_batches.py` | Склеивает батчи разметки тональности в один файл |
| `llm_sentiment_label.py` | Альтернативная разметка тональности через OpenAI API (опционально) |

## Структура папок

```
exports/          — скачанные посты каналов
reports/          — готовые отчёты
  report.md       — сводный
  by_author/      — по каждому автору
artifacts/        — промежуточные файлы (очередь на разметку, батчи)
```
